# AnchorClick instructions

## Project overview

anchor-click is a zero-dependency browser library that makes entire elements clickable by delegating pointer events to an anchor link within. It supports named link targeting, ignored elements, dynamic DOM changes via MutationObserver, and a destroy/re-init lifecycle.

## Project structure

```
src/anchorClick.js       — Source of truth (UMD format)
src/anchorClick.d.ts     — TypeScript definitions (hand-maintained)
build.js                 — Build script (generates dist/)
dist/                    — Generated output (not in git)
  anchorClick.js         — UMD copy
  anchorClick.esm.js     — ESM build
  anchorClick.min.js     — Minified UMD
  anchorClick.min.js.map — Sourcemap for minified build
  anchorClick.d.ts       — TypeScript definitions
tests/                   — Vitest test suite (happy-dom)
index.html               — Demo page (GitHub Pages, served from root)
styles.css               — Demo page styles
eslint.config.mjs        — ESLint 9 flat config
vitest.config.mjs        — Vitest config
CHANGELOG.md             — Release changelog
```

## Key conventions

- **No runtime dependencies.** The library runs in the browser only and must stay dependency-free.
- **UMD source.** `src/anchorClick.js` uses a UMD wrapper (AMD, CommonJS, global). The ESM build is derived from it automatically by `build.js` via factory extraction (string slice).
- **No TypeScript source.** Type definitions live in `src/anchorClick.d.ts` and are copied to `dist/` by `build.js`. Update them manually when the public API changes.
- **No transpilation.** The source targets modern browsers directly — no Babel, no bundler.
- **UMD wrapper is intentionally ES5.** The factory internals use modern JS (const/let, arrow functions, template literals). Do not modernise the UMD wrapper itself.
- **Factory function, not constructor.** `anchorClick()` is called without `new`. It returns a plain object with a `destroy()` method.

## Build

```bash
npm run build     # generates dist/ (UMD, ESM, min, min.js.map, d.ts)
```

The `prepare` script also runs `node build.js`, so `npm install` from a git clone will build automatically.

The ESM build is generated by extracting the factory function from the UMD source via string slice and wrapping it as:
```js
const anchorClick = (factory)(globalThis);
export default anchorClick;
```

## Lint

```bash
npm run lint      # ESLint 9 flat config
```

Config lives in `eslint.config.mjs`. Browser globals for `src/`, Node globals for `build.js`, module + PointerEvent global for `tests/`. `dist/` and `node_modules/` are ignored.

## Test

```bash
npm test          # vitest run (happy-dom environment)
```

Tests live in `tests/anchorClick.test.js`. The UMD source is loaded via `new Function()` with a local `module.exports` so it runs in the happy-dom environment without an import. Tests use `vi.useFakeTimers()` for timing (downUpTime) and `vi.spyOn(window, 'open')` for modifier-key new-tab behaviour.

**Limitations:** happy-dom's pointer event simulation is limited. Tests fire raw `PointerEvent` dispatches on `window`. The `isPrimary` property must be set explicitly (`new PointerEvent('pointerdown', { isPrimary: true, ... })`).

## Release

There are no GitHub Actions workflows. Publishing is done manually:

```bash
npm run build
npm test
npm publish --access public
```

## Public API

The library exports a single factory function:

```js
const instance = anchorClick(options?)
```

Returns an object with a single method: `destroy()`.

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `parent` | `string` | `"data-anchor-target"` | Attribute on the clickable container element |
| `link` | `string` | `"data-anchor"` | Attribute on the target anchor element |
| `ignore` | `string` | `"data-anchor-ignore"` | Attribute to exclude child elements from triggering navigation |
| `clickableClass` | `string` | `"is-clickable"` | Class added to containers that have a matching link |
| `downUpTime` | `number` | `200` | Max ms between pointerdown and pointerup to count as a click |
| `onClick` | `function\|null` | `null` | Callback fired before navigation: `(item, link) => void` |

### Named links

Set matching values on both attributes to designate one link as the primary target when a container has multiple links:

```html
<div data-anchor-target="primary">
  <a href="/page" data-anchor="primary">Primary</a>
  <a href="/other">Other (independent)</a>
</div>
```

### Automatic ignores

These are never treated as item clicks regardless of `ignore`:
- Elements matching `button, input, select, textarea`
- Any `[href]` element that does not carry the `link` attribute (independent links)
- Direct clicks on or inside the `link` element itself (browser handles natively)

## When editing

- Always edit `src/anchorClick.js` and `src/anchorClick.d.ts`, never `dist/` files directly.
- Run `npm run build` after changes to regenerate `dist/`.
- If you change options or the return type, update `src/anchorClick.d.ts`.
- The demo page (`index.html`) loads `src/anchorClick.js` directly (not the built version).
- All pointer event handling uses `pointerdown`/`pointerup`/`pointercancel` — never `click` or `mousedown`.
- `window.open` always passes `'noopener,noreferrer'` as the third argument.
